# DL-04-001: Node/Express statik servis + tarayıcı JS (Fisher-Yates istemcide)

- Tarih: 2026-07-19
- Faz: 4 — Çözüm Analizi
- Mod: AUTOPILOT
- Karar: Ürün, küçük bir Node.js/Express sunucusuyla paketlenir — sunucu YALNIZ `public/` statik dosyaları ve `GET /health` sunar; tüm oyun mantığı (Fisher-Yates karıştırma) vanilla JS ile istemcide çalışır. Docker `node:alpine` tabanlı, tek bağımlılık (express).
- Değerlendirilen alternatifler: B — salt statik (nginx/http-server, sunucu kodu yok); C — SSR/full-stack framework (Next.js/SvelteKit).
- Gerekçe: A, NFR-2 (≤200KB, vanilla), NFR-3 (durumsuz), NFR-7 (~50-70MB imaj), NFR-8 (`/health` first-class route) hepsini karşılar ve coinflip/dice-game SSH-push deploy kalıbıyla birebir uyumludur (sıfır yeni altyapı). B `/health`'i statik dosyaya indirger + deploy emsalinden sapar; C durumsuz/mantığı-istemcide üründe overkill (imaj/build/saldırı yüzeyi NFR ihlali).
- Riskler: express bağımlılığı → `npm audit` Faz 7/9'da izlenir (NFR-3 Critical/High=0). Düşük risk; tek bağımlılık.
- Geri alınabilirlik: Yüksek — oyun mantığı+UI tamamen `public/` içinde statik; sunucu ince kabuk, gerekirse nginx/salt statiğe (B) veya başka runtime'a taşınabilir.
- İnsan onayı: Otomatik
- Varsayım mı?: Hayır — kısıtlardan (istemci-mantık, durumsuz, /health, deploy emsali) doğrudan türedi.
